# HOUSEMAN 시스템 자동화 로드맵

> 핵심 원칙: **시스템이 일정을 이끌고, 사람은 판단만 한다**

---

## 1. 오늘 할 일 대시보드 (Task Driver)

### 목적
로그인하면 "오늘 뭐 해야 하는지" 바로 보인다. 까먹을 수 없다.

### 화면 구성

```
┌─────────────────────────────────────────────┐
│  🔴 미완료 (어제 이전)          3건         │
│  ─────────────────────────────────────────  │
│  ⚡ 오늘 할 일                  8건         │
│  ─────────────────────────────────────────  │
│  📋 이번 주 예정               12건         │
└─────────────────────────────────────────────┘
```

### 할 일 자동 생성 규칙

| 트리거 | 할 일 생성 | 담당자 | 마감 |
|--------|-----------|--------|------|
| 정산일 D-7 | "OO빌 정산서 작성" | 건물 담당자 | 정산일 |
| 정산일 당일 | "OO빌 정산 송금 + 건물주 발송" | 건물 담당자 | 당일 |
| 재계약 D-60 | "OO호 재계약 연락" | 건물 담당자 | D-30까지 |
| 재계약 D-30 (미처리) | "OO호 재계약 미연락 ⚠️" | 건물 담당자 | 즉시 |
| 청구일 도래 | "OO빌 3월분 청구 발송" | 건물 담당자 | 청구일+1 |
| 청구 후 D+3 미납 | "OO호 1차 독촉" | 시스템 자동 | 자동 |
| 청구 후 D+7 미납 | "OO호 2차 독촉" | 시스템 자동 | 자동 |
| 청구 후 D+14 미납 | "OO호 수금 전화 필요" | 수금 담당 | D+14 |
| 순회 예정일 | "OO빌 순회 (마지막: N일 전)" | 외부 담당자 | 예정일 |
| 순회 N일 초과 미실행 | "OO빌 순회 지연 ⚠️" | 외부 담당자 | 즉시 |
| AS 접수 | "OO빌 OO호 AS 처리" | 외부 담당자 | 접수+2일 |
| AS D+3 미처리 | "OO빌 AS 지연 ⚠️ (D+N)" | 외부 담당자 | 즉시 |
| 퇴실 예정 D-7 | "OO호 퇴실 검침 준비" | 외부 담당자 | 퇴실일 |
| 퇴실 완료 | "OO호 퇴실정산서 작성" | 건물 담당자 | 퇴실+1일 |
| 퇴실정산 완료 | "OO호 건물주 연락 + 예치금 정산" | 건물 담당자 | 정산+1일 |
| 공실 N일 경과 | "OO호 공실 N일째 — 홍보 확인" | 건물 담당자 | — |
| 입주 예정 D-1 | "OO호 입주 준비 (열쇠/청소 확인)" | 외부 담당자 | 입주전일 |
| 검침일 | "OO빌 검침" | 외부 담당자 | 검침일 |

### 화면 상세

**필터**: [전체] [내가 담당] [내부] [외부] [수금]
**정렬**: 마감 임박순 (기본) / 건물별 / 유형별
**상태**: ⬜ 미완료 → ✅ 완료 (클릭으로 전환)
**미완료 누적**: 어제 이전 미완료 → 🔴 빨간 배지로 상단 고정

### 데이터 연결
- 기존: 정산 타임라인, 재계약 만기, 입퇴실 일정, AS 목록, 순회 기록
- 신규: `tasks` 테이블 (자동 생성 + 수동 추가 가능)
- 담당자: `buildingData[건물].managers` 에서 자동 배정

### 알림
- 매일 아침 9시: "오늘 할 일 N건" 요약 (향후 카톡/문자 연동 가능)
- 미완료 D+1: 빨간색 승격
- 미완료 D+3: 대표에게 에스컬레이션 표시

## 2. 수익 대시보드 (Profit Dashboard)

### 목적
건물별로 얼마 벌고 있는지 한눈에 본다. "이 건물 계속 맡을 가치가 있나?" 판단 근거.

### 화면 구성

```
┌──────────────────────────────────────────────┐
│  2026년 3월 수익 현황                        │
│                                              │
│  총 매출: 32,450,000원                       │
│  총 비용: 18,200,000원                       │
│  순이익:  14,250,000원  (마진율 43.9%)       │
└──────────────────────────────────────────────┘

┌──────────────────────────────────────────────┐
│  건물별 수익 순위                             │
│                                              │
│  1. 스타빌      월세수수료 285만 + 마진 42만  │
│                 비용배분 -80만 = 순이익 247만  │
│  2. 아페이론    월세수수료 312만 + 마진 38만  │
│                 비용배분 -80만 = 순이익 270만  │
│  ...                                         │
│  ─── 수익률 하위 ───                         │
│  28. 풍림빌딩   매출 0 = ⚠️ 재검토 필요      │
└──────────────────────────────────────────────┘
```

### 수익 항목 (건물별 자동 계산)

**매출 (수입)**
| 항목 | 계산 방식 | 데이터 소스 |
|------|-----------|-------------|
| 관리수수료 (%) | 월세합계 × 수수료율 | 정산 데이터 |
| 관리수수료 (정액) | 월정액 | settlementMaster |
| 관리수수료 (월급형) | 월급액 | settlementMaster |
| 전기/가스 마진 | 호실수 × 5,000원 | 임차인 수 |
| 인터넷/수도 정액 마진 | 징수액 × 40% | 청구 데이터 |
| 퇴실청소 마진 | 청구액 - 업체지급액 | 퇴실 데이터 |
| 수리/공사 마진 | 청구액 - 업체지급액 | AS 데이터 |
| 중개수수료 | 계약 시 수수료 | 입주 데이터 |
| 위약금 귀속분 | 7일 패널티 (건물별 설정) | 퇴실 데이터 |
| 청소업체 10% | 청소매출 × 10% (근생) | 별도 입력 |

**비용 (지출)**
| 항목 | 계산 방식 |
|------|-----------|
| 인건비 배분 | 총 인건비 ÷ 건물수 (또는 세대수 비례) |
| 차량/교통비 | 건물별 방문횟수 비례 |
| AS 지출 (자사부담) | 무상 수리비 |
| 기타 지출 | 정산 공제내역 중 자사부담분 |

### 뷰 옵션
- **월간**: 이번 달 건물별 수익
- **연간**: 12개월 추이 그래프
- **건물 상세**: 클릭 → 해당 건물 수익 내역
- **수익률 하위 경고**: 순이익 마이너스 or 하위 20% 건물 자동 표시

### 입력 필요 항목 (한 번만 설정)
- 건물별 마진 항목 ON/OFF (전기마진 있는 건물 / 없는 건물)
- 퇴실청소 업체 지급 기본액
- 인건비 배분 방식 선택 (균등 / 세대수비례 / 수동)
- 수리/공사 마진율 기본값

## 3. 자동 청구 + 독촉 시스템

### 목적
청구 → 확인 → 발송 → 미납감지 → 독촉 1~3차까지 시스템이 자동. 사람은 4차(전화)만.

### 프로세스 흐름

```
[매월 청구일 도래]
    ↓
[시스템] 청구서 자동 생성 (호실별 금액 계산)
    ↓
[담당자] "오늘 할 일"에서 확인 → 클릭 한 번으로 발송
    ↓
[시스템] 문자/카톡 자동 발송
    ↓
[시스템] 입금 자동 감지 (입출금 데이터 매칭)
    ↓ 미입금 시
[D+3] 시스템 → 1차 독촉 자동 발송 (안내형)
    ↓ 미입금 시
[D+7] 시스템 → 2차 독촉 자동 발송 (경고형)
    ↓ 미입금 시
[D+14] 시스템 → 3차 독촉 자동 발송 (최종형)
    ↓ 미입금 시
[D+14] "오늘 할 일"에 "OO호 수금 전화 필요" 생성
    ↓
[수금 담당] 전화 → 결과 기록
```

### 수금 전용 화면

```
┌──────────────────────────────────────────────┐
│  📞 오늘 전화할 사람  (7건)                  │
│                                              │
│  🔴 스타빌 403 송예준  30만원  D+18          │
│     "분납 약속 → 3/20까지 15만원"            │
│     [전화완료] [분납약속] [방문예정]          │
│                                              │
│  🟡 아페이론 103 김지은  50만원  D+14         │
│     [전화완료] [부재중] [문자발송]            │
│                                              │
│  ── 자동 처리 중 (전화 불필요) ──            │
│  ⚪ 스타빌 202 이지람  40만원  D+5 (2차독촉) │
│  ⚪ 아페이론 203 이준우 70만원  D+3 (1차독촉) │
└──────────────────────────────────────────────┘
```

### 독촉 메시지 자동 발송 규칙
- 1차 (D+3): "OO님, 3월분 월세 OO만원이 미납되었습니다. 확인 부탁드립니다."
- 2차 (D+7): "OO님, 미납금 OO만원 안내드립니다. 3일 내 납부 부탁드립니다."
- 3차 (D+14): "OO님, 최종 안내입니다. 미납 시 연체료가 부과됩니다."
- 4차 (D+14): **사람에게 넘김** — "오늘 할 일"에 등록

### 데이터 연결
- 청구 금액: 기존 청구 시스템
- 입금 매칭: 입출금 데이터 (건물+호실+금액 매칭)
- 독촉 템플릿: 기존 DunningTemplateSettings 연동
- 수금 기록: 통화 결과 + 약속 내용 저장

## 5. 순회 스케줄 + 동선 제안

### 목적
"언제 어디를 가야 하는지" 시스템이 알려준다. 까먹지 않고, 효율적으로 돈다.

### 자동 스케줄링 규칙

```
[매주 월요일 자동 생성]
    ↓
이번 주 외부 업무 수집:
  - AS 미처리 건 (우선순위 높음)
  - 검침 예정 건물
  - 순회 주기 도래 건물
  - 입퇴실 예정 건물
    ↓
권역별 그룹핑:
  - 관악/동작 권역
  - 강남/서초 권역
  - 중구/마포 권역
  - 기타
    ↓
동선 제안:
  "화요일: 관악권 — 스타빌(AS) → 아페이론(순회) → 제이앤제이(검침)"
  "수요일: 중구권 — 상건빌딩(순회) → 미진빌딩(AS) → 유석빌딩(검침)"
```

### 동선 제안 화면

```
┌──────────────────────────────────────────────┐
│  📍 이번 주 동선 제안                        │
│                                              │
│  화요일 (3/19) — 관악 권역  4건              │
│  ┌─ 09:30  스타빌 (AS: 201호 수전)          │
│  │  10:30  아페이론 (순회: 42일 경과)        │
│  │  11:30  제이앤제이 (검침)                 │
│  └─ 13:00  포유빌 (입주 체크: 301호)         │
│                                              │
│  수요일 (3/20) — 중구 권역  3건              │
│  ┌─ 10:00  상건빌딩 (순회: 35일 경과)        │
│  │  11:30  미진빌딩 (AS: 302호 누수)         │
│  └─ 13:00  유석빌딩 (검침)                   │
│                                              │
│  ⚠️ 미배정: 어반그레이 (남양주) — 순회 50일  │
│     → 별도 일정 필요                          │
└──────────────────────────────────────────────┘
```

### "근처 건물" 자동 제안
- AS 접수 시 → "이 건물에서 반경 1km 내 순회 예정 건물: OO빌 (순회 38일 경과)"
- 검침 방문 시 → "같은 권역 AS 미처리: OO빌 OO호"
- 순회 완료 시 → "근처 OO빌도 순회 예정입니다. 같이 하시겠습니까?"

### 건물 위치 데이터
- 기존 `buildingData[건물].address` 활용
- 주소 → 권역 자동 분류 (구 단위)
- 향후: 좌표 기반 거리 계산 (카카오맵 API)

### 순회 지연 경고
- 순회 주기 (월1회/월2회 등) 대비 경과일 자동 계산
- 주기 초과 시 → "오늘 할 일"에 빨간색 등록
- 대시보드에 "순회 지연 건물 TOP 5" 표시

## 6. 건물주 월간 보고서 자동화

### 목적
매달 건물주에게 "당신 건물 이렇게 관리하고 있습니다" 보고서를 자동 발송. 신뢰 + 해지 방어 + 마케팅 소재.

```
┌──────────────────────────────────────────────┐
│  ┌────────────────────────────────────┐      │
│  │  스타빌 월간 관리 보고서           │      │
```

### 자동 생성 프로세스

```
[매월 말일 + 3일 (정산 완료 후)]
    ↓
[시스템] 건물별 데이터 자동 수집:
  - 입주 현황 (activeTenants)
  - 공실 현황 (activeVacancies)
  - 수금률 (billingHistory + roomBalances)
  - 순회 사진 (순회 기록)
  - AS 처리 내역 (AS 데이터)
  - 정산 요약 (settlement 데이터)
    ↓
[시스템] 보고서 자동 생성 (웹 페이지 / PDF)
    ↓
[담당자] "오늘 할 일"에 "OO빌 건물주 보고서 확인/발송" 생성
    ↓
[담당자] 검토 → 수정 가능 → 발송
    ↓
[시스템] 건물주 이메일/카톡 발송
```

### 건물주별 설정
- 발송 수단: 이메일 / 카톡 / 둘 다 / 미발송
- 보고서 상세도: 전체 / 요약만 / 정산만
- 특이사항 코멘트: 담당자가 한 줄 추가 가능

### 마케팅 활용
- 보고서 샘플 → 블로그/유튜브 콘텐츠
- "이런 보고서가 매달 갑니다" → 신규 건물주 영업 자료
- 건물주 전용 웹 페이지 (로그인 후 열람) → 향후 확장

## 7. 정산서 완전 자동 생성

### 목적
말일에 몰리는 정산서 작성을 "검토만 하면 되는" 수준으로 자동화.

### 현재 vs 목표

| 단계 | 현재 | 목표 |
|------|------|------|
| 금액 계산 | ✅ 자동 | ✅ 유지 |
| 호실별 상세 | ✅ 화면 표시 | ✅ 유지 |
| 입퇴실 반영 | ✅ 자동 | ✅ 유지 |
| 공제내역 | ⚠️ 수동 입력 | ⚠️ 유지 (AS연동 강화) |
| 엑셀/PDF 출력 | ⚠️ 미리보기만 | ✅ 원클릭 다운로드 |
| 건물주 발송 | ❌ 수동 | ✅ 이메일 자동 발송 |
| 출납 등록 | ⚠️ 버튼 클릭 | ✅ 발송 시 자동 등록 |
| 세금계산서 | ❌ 별도 | ⚠️ 발행 알림 + 연동 안내 |

### 자동화 프로세스

```
[정산일 D-7]
    ↓
[시스템] 정산서 초안 자동 생성
  - 월세/관리비 자동 계산
  - 입퇴실 일할 자동 반영
  - 이번 달 AS 중 공제 대상 자동 연결
  - 이전 미수금 자동 반영
    ↓
[담당자] "오늘 할 일"에 "OO빌 정산서 검토"
    ↓
[담당자] 검토 → 공제내역 수정/추가 → 확정
    ↓
[정산일 당일]
[담당자] "오늘 할 일"에 "OO빌 정산 발송"
    ↓
[시스템] 원클릭으로:
  1. 엑셀/PDF 생성
  2. 건물주 이메일 발송
  3. 출납관리에 자동 등록
  4. VAT 건물은 "세금계산서 발행" 할 일 추가
```

### 공제내역 자동 연결
- AS 처리 완료 + "건물주 부담" 표시된 건 → 공제내역에 자동 추가
- 공용전기/인터넷 등 정기 지출 → 매월 자동 반복
- 담당자는 **추가/삭제만** 하면 됨

### 엑셀 호환
- 기존 건물주가 받아보던 엑셀 양식 유지
- 건물별 커스텀 양식 (1시트/2시트/층별분리 등) 설정 가능
- 처음 한 번 설정 → 매월 자동 적용

## 구현 순서 요약

```
Phase 1 (즉시)
  ├─ 1. 오늘 할 일 대시보드
  └─ 2. 수익 대시보드

Phase 2 (Phase 1 확인 후)
  ├─ 3. 자동 청구/독촉
  └─ 4. 모바일 현장 입력

Phase 3 (Phase 2 확인 후)
  ├─ 5. 순회 동선 제안
  └─ 6. 건물주 보고서

Phase 4 (마무리)
  └─ 7. 정산서 자동 생성
```

### 각 Phase 완료 기준
- Phase 1: 로그인하면 할 일이 보이고, 건물별 수익이 보인다
- Phase 2: 청구~수금이 자동 흐르고, 현장에서 입력이 끝난다
- Phase 3: 외부팀 동선이 효율적이고, 건물주가 보고서를 받는다
- Phase 4: 말일 정산이 "검토+클릭"으로 끝난다

6. "정산서 5분 만에 끝납니다" — 자동 정산
7. "30개 건물, 10명이 관리하는 비결" — 전체 시스템 소개
