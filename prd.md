# HOUSEMAN — 부동산 관리 시스템 PRD

## 1. 제품 개요

### 1.1 제품명
**HOUSEMAN (하우스맨)** — 건물관리 시스템

### 1.2 목적
다수의 임대 건물을 관리하는 부동산 관리업체가 세입자, 수금, AS, 공실, 공과금, 순회 등 건물 운영에 필요한 모든 업무를 하나의 시스템에서 통합 관리할 수 있도록 한다.

### 1.3 대상 사용자
| 역할 | 설명 |
|------|------|
| **관리자 (admin)** | 건물 관리 전반을 담당하는 내부 직원 (대표, 부장, 차장, 과장, 사원) |
| **건물주 (owner)** | 소유 건물의 현황·수익만 열람하는 건물 소유자 |
| **청소팀 (cleaning)** | 계약/퇴실 일정과 순회 관리만 접근하는 청소 담당자 |
| **홈페이지 (homepage)** | 공실 매물 정보를 일반 고객에게 노출하는 공개 뷰 |

### 1.4 핵심 가치
- **한눈에 보는 운영 현황**: 대시보드에서 연체·AS·공실·수금·캘린더를 실시간 파악
- **역할 기반 접근 제어**: 관리자/건물주/청소팀/홈페이지 뷰를 분리하여 필요한 정보만 노출
- **담당자별 건물 배정**: 직원마다 담당 건물을 지정하여 데이터를 필터링
- **모바일 반응형**: 현장에서 모바일로 즉시 접근 가능

## 4. 기능 상세

### 4.1 인증 및 접근 제어

#### 4.1.1 로그인
- 연락처(ID) + 비밀번호로 로그인
- 테스트 계정 8개 제공 (클릭 시 자동 입력)
- 로그인 후 담당자 역할에 따라 접근 가능 메뉴 결정

#### 4.1.2 역할별 뷰 모드
| 뷰 모드 | 접근 메뉴 |
|---------|----------|
| 관리 (admin) | 대시보드, 캘린더, 건물·호실, 세입자·계약, 공실관리, 수금관리, 공과금청구, 입출금관리, AS관리, 순회관리, 담당자관리 |
| 건물주 (owner) | 건물주 전용 대시보드 (소유 건물 현황, 수익, 공실, 계약갱신) |
| 청소 (cleaning) | 계약/퇴실 캘린더, 순회 관리 |
| 홈페이지 (homepage) | 공실 매물 공개 페이지 |

#### 4.1.3 담당자별 건물 필터링
- 직원별 담당 건물이 지정되어 있으며, 본인 담당 건물의 데이터만 표시
- `general` 역할은 전체 건물 접근 가능

### 4.3 건물·호실 관리 (Buildings)

#### 4.3.1 건물 목록
- 42개 건물 카드형 리스트
- 건물별 호실 수, 입주율, 수수료 체계(% 또는 정액) 표시
- 한글 초성 검색 지원 (예: "ㄷ" → "다존하우스")
- 건물 유형 필터 (전체/주택)

#### 4.3.2 건물 상세 (Building Detail)
- 층별 호실 현황 시각화 (정상/연체/공실 색상 구분)
- 호실 클릭 시 세입자 상세 정보 표시
  - 기본 정보: 이름, 연락처, 계약유형, 보증금, 월세, 관리비
  - 계약 정보: 납부일, 만료일, 연체 금액
  - 이전 세입자 이력
  - 호실 마스터 데이터 (면적, 옵션)
  - 호실 유형 변경 (단기 ↔ 일반)
- 건물 수수료 정보 표시

## 7. 비즈니스 로직

### 7.1 수수료 계산
- **비율제 (pct)**: 월 임대료 × 수수료율
- **정액제 (fixed)**: 월 고정 수수료 금액

### 7.2 연체 판단
- 납부일 기준 미납 시 "연체" 상태
- 연체 금액 및 연체 일수 자동 계산

### 7.3 공과금 청구 흐름
1. 건물별 검침 데이터 입력 (전기/가스 지침)
2. 사용량 계산 (당월지침 - 전월지침)
3. 월세 + 관리비 + 수도 + 케이블 + 전기 + 가스 합산
4. 확인 → 발송 2단계 상태 관리

### 7.4 잔액 관리
- `roomBalances`: 호실별 잔액 = 청구 누적 - 입금 누적
- `addBilling()`: 청구 시 잔액 증가
- `addDeposit()`: 입금 시 잔액 감소 (최소 0)

## 8. 현재 한계 및 향후 개선 방향

### 8.1 현재 한계 (프로토타입)
| 항목 | 상태 |
|------|------|
| 데이터 저장 | 클라이언트 메모리 (새로고침 시 초기화) |
| 백엔드 API | 미구현 (정적 데이터) |
| 인증 | 평문 비밀번호, 세션 미관리 |
| 파일 구조 | 단일 파일 6,500줄 (모듈 분리 필요) |
| 이미지 | 플레이스홀더만 (실제 업로드 미구현) |
| 알림 | 하드코딩 배지 (실시간 알림 미구현) |
| 날짜 | 2026-02-22 고정 (동적 날짜 미적용) |

### 8.2 향후 개선 방향
1. **백엔드 연동**: REST API 또는 Supabase/Firebase 연결로 데이터 영속화
2. **모듈 분리**: 컴포넌트/페이지/유틸리티를 별도 파일로 분리
3. **상태 관리**: Context API 또는 Zustand 도입
4. **라우팅**: React Router 도입
5. **인증 강화**: JWT 기반 인증, 역할 기반 권한 미들웨어
6. **실시간 알림**: WebSocket 기반 연체/AS 알림
7. **사진 업로드**: 클라우드 스토리지 연동
8. **리포트**: 월간/분기별 수익 보고서 자동 생성
9. **SMS/카톡 연동**: 청구서·연체 알림 자동 발송
10. **다국어 지원**: 외국인 세입자를 위한 영어 지원
