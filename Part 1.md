# Phase 2 Part 1 Round 1 — CalendarPage 기능 통합 (Supabase 무관)

Phase 2 Part 1 Round 1을 비유하면 — **기존 캘린더 벽에 새 메모판과 경고등을 추가한 것이다.** 캘린더 구조(벽) 자체는 그대로 두고, 대표님이 현장에서 필요해서 만든 기능들을 하나씩 옮겨 붙였다.

## 뭘 한 건가

**분석** — 대표님 CalendarPage(2,668줄)와 우리 calendar/ 디렉토리(15개 파일)를 비교해서, 대표님이 추가/변경한 기능 15개를 식별했다. 이 중 Supabase(외부 DB)와 무관한 8개를 먼저 반영했다.

**wobble 애니메이션** — 퇴실 체크리스트에서 미완료 항목이 있으면 흔들리는 효과. "이거 아직 안 했어요"를 시각적으로 알려주는 것이다.

**waterFee 필드명 변경** — SMS 템플릿에서 `water`를 `waterFee`로 바꿨다. 임차인에게 보내는 문자에서 "수도요금" 항목이 제대로 표시되도록 수정.

**Toast 알림** — 이벤트 저장/삭제 시 화면 하단에 "저장 완료", "삭제 완료" 같은 알림이 뜬다. 기존에는 아무 피드백 없이 그냥 처리됐는데, 이제 직원이 "잘 됐구나"를 확인할 수 있다.

**단기/일반 퇴실 분기** — 퇴실 처리할 때 "이 호실이 단기 임대인가, 일반 임대인가"를 자동으로 판별해서 다른 체크리스트를 보여준다. 단기는 청소/사진만, 일반은 계량기 검침/보증금 정산까지 포함. 대표님이 현장에서 "단기랑 일반이 절차가 다른데 같은 화면이라 헷갈린다"는 문제를 해결한 것이다.

**예금주 불일치 경고** — 보증금 환불할 때 계약서의 임차인 이름과 환불 계좌 예금주가 다르면 경고를 띄운다. 잘못된 계좌로 송금하는 사고를 막는 안전장치.

**자동삭제 보강** — 퇴실 이벤트가 완료 처리되면 자동으로 삭제되는 로직이 있는데, "외부팀 체크가 끝난 건"은 삭제하지 않도록 예외를 추가했다. 기록이 남아야 나중에 정산 근거가 되니까.

**건물목록 소스 변경** — 캘린더에서 건물 필터를 선택할 때, 기존 고정 데이터가 아니라 실제 등록된 건물 목록에서 읽도록 변경. 건물이 새로 추가되면 캘린더 필터에도 자동으로 나타난다.

**SendLinkModal (신규)** — 퇴실 링크를 임차인에게 보낼 때, 등록된 번호 말고 다른 번호로도 보낼 수 있는 모달. 클립보드 복사 또는 SMS 전송 지원. "임차인이 번호를 바꿨는데 시스템에 아직 반영 안 됐을 때" 쓰는 기능이다.

## 왜 이 8개를 먼저 했나

이 기능들은 전부 **Supabase(외부 DB)와 무관**하다. 우리 기존 코드에 그냥 얹으면 되는 것들이라 안전하게 먼저 처리할 수 있었다. 나머지 7개(사진 업로드, 외부팀 체크, 직접입력 등)는 Supabase를 직접 호출하기 때문에, feature flag로 "API 모드 vs Supabase 모드"를 분기하는 작업이 필요하다. 이건 Round 2에서 처리한다.

## 구조

캘린더 디렉토리에 새 파일 1개 추가 (SendLinkModal.tsx), 기존 파일 6개 수정. **디렉토리 구조는 변경 없음** — 우리가 분해한 15개 파일 체계가 그대로 유지된다.
